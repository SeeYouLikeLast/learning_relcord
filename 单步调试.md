有时需要打开Cmake删除缓存并重新配置，之后再进行build，之后就可以进行调试


下是详细的步骤指南：
第一步：准备工作 (安装必备工具)

确保你的系统已经安装了编译器和调试器：
Bash

sudo apt update
sudo apt install build-essential cmake gdb

第二步：安装 VS Code 插件

在 VS Code 的扩展商店（Extensions）中安装以下两个核心插件：

    C/C++ (Microsoft) - 提供 IntelliSense 和调试功能。

    CMake Tools (Microsoft) - 提供 CMake 项目的自动化管理。

方法一：使用 CMake Tools 插件 (最快上手)

这种方法不需要你手写复杂的配置文件，适合绝大多数场景。
1. 准备代码

假设你有一个简单的 main.cpp 和 CMakeLists.txt。

CMakeLists.txt 示例:
CMake

cmake_minimum_required(VERSION 3.0)
project(MyDebugProject)

# 确保生成调试符号 (如果不使用 CMake Tools 的 Variant 管理，这行很重要)
set(CMAKE_BUILD_TYPE Debug)

add_executable(my_app main.cpp)

2. 配置项目 (Configure)

    用 VS Code 打开项目文件夹。

    VS Code 可能会弹出右下角提示 "Configure project?"，点击 Yes。

    如果没有弹出，按下 Ctrl + Shift + P，输入 CMake: Configure。

    选择编译器 (Kit): 选择检测到的 GCC (例如 GCC 9.x.x x86_64-linux-gnu)。

3. 设置为 Debug 模式 (关键步骤)

为了能单步调试，必须确保编译时包含调试符号，并且没有开启高等级优化。

    点击 VS Code 底部状态栏的 CMake: [Debug/Release] 按钮（通常在左下角）。

    在弹出的列表中选择 Debug。

4. 编译 (Build)

    点击底部状态栏的 Build 按钮，或者按 F7。

    确保输出窗口显示 Build finished with exit code 0。

5. 开始调试

    打开 main.cpp，在行号左侧点击，打上红点（断点）。

    方法 A (快捷键): 直接按 Ctrl + F5。

    方法 B (状态栏): 点击底部状态栏虫子图标旁边的 Debug 按钮。

此时，程序会自动启动 GDB，并在你的断点处停下。你可以使用顶部的调试工具栏进行：

    F10: 单步跳过 (Step Over)

    F11: 单步进入 (Step Into)

    Shift + F11: 单步跳出 (Step Out)
